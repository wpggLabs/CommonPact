# CommonPact Brand and Web System

The public design should look like a protocol specification and civic infrastructure project, not a generic startup landing page.

## Visual language

- near-black field, warm off-white text, signal green for protocol paths, cobalt for profile layers;
- square geometry, thin rules, visible identifiers, version labels, and system diagrams;
- editorial typography with monospace metadata;
- no stock illustrations, fake product screenshots, glass cards, decorative blobs, or invented adoption metrics;
- motion is optional and never required for understanding;
- every visual must explain architecture, lifecycle, responsibility, or status.

## Family relationship

PactRide and CommonPact should share disciplined dark surfaces, technical labels, strong contrast, and honest status language. CommonPact uses a broader network-grid motif because it is the base protocol; PactRide remains transportation-specific.

## Mark

`CP/00` represents the CommonPact core before any domain profile. Profile labels use `CP/01`, `CP/02`, and so on only as visual navigation, not protocol identifiers.

## Voice

- direct, concrete, and skeptical;
- “no mandatory platform owner,” not “no middlemen ever”;
- documented vision separated from implementation evidence;
- no claims of decentralization, safety, legality, or adoption without measurements.
