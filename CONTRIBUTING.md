# Contributing

High-value contributions include falsifiable criticism, independent implementation, valid/invalid fixtures, privacy or security analysis, profile-domain review, accessible workflow documentation, primary-source prior art, and reproducible transport measurements.

Before proposing a core feature, explain why it belongs in the core instead of one profile, how it works in at least two distinct domains, signer and lifecycle effects, privacy and security consequences, migration, and test vectors.

Keep pull requests focused. Update schemas, examples, status, and website claims when semantics change.

Do not submit real personal trip, rental, delivery, donation, identity, payment, or health data.
