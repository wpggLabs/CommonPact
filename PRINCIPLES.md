# Principles

1. **Protocol before platform** — CommonPact defines interoperable semantics. No official application is required.
2. **No mandatory owner** — the base protocol requires no single company, relay, registry, payment provider, identity provider, or operator.
3. **No protocol tax** — creating, accepting, or validating a pact requires no commission payable to CommonPact.
4. **Transparent optional services** — paid services are allowed, but dependencies, fees, ranking influence, and data handling should be disclosed.
5. **User-held identity and records** — users control authorization keys and can export evidence.
6. **Profiles carry domain meaning** — generic abstractions must not erase safety, legal, accessibility, or operational requirements.
7. **Privacy by minimization** — public discovery exposes only what is needed to find a potential counterparty.
8. **Required-party agreement** — agreement exists only when the profile-required parties authorize identical canonical terms.
9. **Claims are not truth** — signatures prove key authorization, not physical events, identity, safety, payment, legality, or fairness.
10. **Conflict is preserved** — valid competing evidence is not silently discarded because one event arrived first.
11. **Transport independence** — protocol meaning does not depend on one carrier.
12. **Local policy, shared evidence** — clients may evaluate evidence differently while preserving provenance.
13. **Interoperability requires evidence** — one reference implementation is not proof of a protocol.
14. **Progressive decentralization** — begin with replaceable internet infrastructure; add direct and mesh capabilities only when measured.
15. **Forkability and succession** — the specification must survive the original founder.
