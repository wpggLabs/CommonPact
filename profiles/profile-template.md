# Profile Template

A profile proposal should include the following files or equivalent sections:

- `README.md` — purpose, roles, non-goals, status, and complete overview;
- `LIFECYCLE.md` — aggregate and thread state machines and signer matrix;
- `TERMS_AND_EVIDENCE.md` — canonical terms, activation, claims, receipts, disputes;
- `PRIVACY_ACCESSIBILITY_AND_ABUSE.md` — data classification and human-risk analysis;
- `RESPONSIBILITY_BOUNDARIES.md` — clients, operators, optional providers, authorities;
- `profile-manifest.json` — machine-readable identity and artifact paths;
- `schemas/` — strict event and extension schemas;
- `test-vectors/` — valid and invalid fixtures;
- a complete fictional transcript under `examples/` or the profile directory.

The package must answer every requirement in `../PROFILE_STANDARD.md`.
